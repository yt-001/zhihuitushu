# 智慧图书阅读平台 — 管理端系统设计方案

---

## 1 系统概述

### 1.1 项目背景

智慧图书阅读平台是一个面向读者的在线阅读服务系统，目前已实现图书管理、用户阅读、AI推荐等核心功能。为便于系统运营和维护，需要开发一套简洁高效的管理端系统。

### 1.2 设计目标

- **简洁实用**：采用单一管理员模式，无需复杂权限系统
- **统一技术栈**：复用现有技术架构（Spring Boot + Vue 3 + MySQL + MyBatis-Plus）
- **快速开发**：基于现有代码扩展，减少重复工作
- **易于维护**：代码结构清晰，接口规范统一

### 1.3 功能范围

| 模块 | 功能点 |
|------|--------|
| 管理员认证 | 登录、登出、密码修改 |
| 图书管理 | 图书列表、新增、编辑、删除、上下架 |
| 分类管理 | 分类列表、新增、编辑、删除 |
| 用户管理 | 用户列表、查看详情、禁用/启用 |
| 数据统计 | 用户统计、图书统计、阅读统计 |
| 系统设置 | 管理员信息、系统参数配置 |

## 3 功能模块设计

### 3.1 管理员认证模块

#### 功能描述

管理员的登录、登出和密码管理功能。

#### 功能点

| 功能 | 说明 |
|------|------|
| 登录 | 管理员使用用户名/密码登录，获取JWT令牌 |
| 登出 | 清除登录状态，使令牌失效 |
| 密码修改 | 管理员修改自己的登录密码 |

#### 业务流程

```
登录流程：
1. 输入用户名和密码
2. 后端验证身份
3. 生成JWT令牌存入Cookie
4. 返回管理员基本信息
5. 跳转到管理后台首页
```

## 4 数据库设计

### 4.1 管理员表（admins）

```sql
-- 管理员表
CREATE TABLE IF NOT EXISTS admins (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '管理员ID',
    username VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
    password VARCHAR(255) NOT NULL COMMENT 'BCrypt加密后的密码',
    nickname VARCHAR(50) NULL COMMENT '昵称',
    avatar VARCHAR(500) NULL COMMENT '头像URL',
    email VARCHAR(100) NULL COMMENT '邮箱',

    last_login_time DATETIME NULL COMMENT '最后登录时间',
    last_login_ip VARCHAR(50) NULL COMMENT '最后登录IP',

    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_username (username),
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='管理员表';
```

## 5 API接口设计概览

### 5.1 接口路由规则

管理端API统一使用 `/api/admin` 前缀：

```
/api/admin/auth/*      # 认证相关
/api/admin/books/*     # 图书管理
/api/admin/categories/*# 分类管理
/api/admin/users/*     # 用户管理
/api/admin/stats/*     # 数据统计
/api/admin/settings/*  # 系统设置
```

### 5.2 认证机制

- 复用现有JWT认证机制
- 管理员认证使用独立的Cookie名称：`admin_token`
- 通过自定义注解 `@AdminAuth` 标识需要管理员权限的接口

## 7 安全设计

### 7.1 认证安全

- 使用BCrypt加密存储管理员密码
- JWT令牌有效期设置为2小时
- 登录失败超过5次锁定账号15分钟
- 记录管理员登录日志（IP、时间、设备）

### 7.2 接口安全

- 所有管理端接口需要验证管理员身份
- 敏感操作（删除、修改状态）记录操作日志
- 接口请求频率限制（100次/分钟）

### 7.3 前端安全

- 路由守卫验证登录状态
- 敏感信息不存储在localStorage
- XSS防护（Vue自动转义）
- CSRF防护（Cookie SameSite）

## 9 部署方案

### 9.1 部署架构

```
┌─────────────────────────────────────────────────────────────┐
│                        Nginx                                │
│           (反向代理 + 静态资源)                              │
└─────────────────────────────────────────────────────────────┘
              │                           │
              ▼                           ▼
┌─────────────────────────┐   ┌─────────────────────────────┐
│    前端静态资源          │   │      后端服务               │
│  (用户端 + 管理端)       │   │   Spring Boot :8080        │
└─────────────────────────┘   └─────────────────────────────┘
                                          │
                    ┌─────────────────────┼─────────────────────┐
                    ▼                     ▼                     ▼
          ┌─────────────────┐   ┌─────────────────┐   ┌─────────────────┐
          │     MySQL       │   │     Redis       │   │    腾讯云COS    │
          │   (主从复制)     │   │   (缓存/会话)   │   │   (文件存储)    │
          └─────────────────┘   └─────────────────┘   └─────────────────┘
```

### 9.2 Nginx配置

```nginx
server {
    listen 80;
    server_name example.com;

    # 用户端
    location / {
        root /var/www/zhihuitushu-frontend;
        try_files $uri $uri/ /index.html;
    }

    # 管理端
    location /admin {
        root /var/www/zhihuitushu-frontend;
        try_files $uri $uri/ /admin/index.html;
    }

    # API代理
    location /api {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```
