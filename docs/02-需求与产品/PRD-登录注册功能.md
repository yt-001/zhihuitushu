# 智慧图书阅读项目 — 登录注册功能 PRD

---

## 1 执行摘要

### 1.1 问题陈述

当前智慧图书阅读平台缺乏用户认证体系，无法实现用户个性化阅读推荐、阅读进度同步、书架管理等核心功能，同时无法保障用户数据安全与隐私保护。

### 1.2 解决方案

构建完整的用户登录注册系统，支持手机号/邮箱注册登录、验证码验证、密码找回等功能，采用HttpOnly Cookie + JWT令牌方案实现安全认证，集成Element Plus组件库构建统一风格的登录注册页面。

### 1.3 成功标准

| 指标 | 目标 |
|------|------|
| 注册成功率 | ≥ 95% |
| 登录响应时间 | ≤ 500ms |
| 验证码发送成功率 | ≥ 99% |
| 用户满意度 | ≥ 85% |
| 安全合规 | 通过OWASP安全扫描，无高危漏洞 |

## 2 用户体验与功能

### 2.1 用户角色

| 角色 | 特征 | 登录注册需求 |
|------|------|-------------|
| **普通读者** | 18-59岁，熟悉智能手机操作 | 手机号/邮箱快捷注册登录 |
| **青少年学生** | 6-18岁，家长管控 | 家长授权注册，内容过滤 |
| **老年人** | 60岁以上，视力下降 | 大字体界面，简化操作流程 |
| **视障用户** | 视力障碍 | 屏幕朗读支持，语音交互 |

### 2.2 用户故事

#### 故事1：用户注册

**As a** 新用户，**I want to** 通过手机号/邮箱注册账号，**so that** 我可以使用平台的个性化阅读服务。

**验收标准：**
- 支持手机号和邮箱两种注册方式
- 注册表单包含：账号、密码、确认密码、验证码
- 密码强度校验：长度≥8位，包含大小写字母和数字
- 验证码有效期5分钟，支持重新发送（60秒冷却）
- 注册成功后自动登录并跳转首页
- 已注册账号提示"该账号已注册，请直接登录"

#### 故事2：用户登录

**As a** 已注册用户，**I want to** 通过账号密码登录平台，**so that** 我可以访问我的书架和阅读记录。

**验收标准：**
- 支持手机号和邮箱登录
- 登录表单包含：账号、密码、记住我选项
- "记住我"勾选后7天内自动登录
- 密码错误3次后锁定账号5分钟
- 登录失败显示明确错误提示（账号不存在/密码错误）
- 登录成功后JWT令牌存入HttpOnly Cookie

#### 故事3：密码找回

**As a** 忘记密码的用户，**I want to** 通过验证码找回密码，**so that** 我可以重新登录平台。

**验收标准：**
- 支持通过手机号/邮箱找回密码
- 验证步骤：输入账号 → 获取验证码 → 设置新密码
- 新密码与原密码不同时方可设置
- 密码找回成功后提示重新登录

#### 故事4：用户退出

**As a** 已登录用户，**I want to** 安全退出登录，**so that** 保护我的账号安全。

**验收标准：**
- 退出登录后清除Cookie中的JWT令牌
- 退出后跳转至登录页面
- 清除本地存储的用户信息

### 2.3 非目标

- 暂不支持第三方登录（微信/QQ/支付宝）
- 暂不支持多设备同时登录管理
- 暂不支持账号注销功能

### 3.2 集成点

| 集成项 | 说明 |
|--------|------|
| **数据库** | MySQL存储用户信息（id、username、password、email、phone、status、created_at、updated_at） |
| **缓存** | Redis存储验证码（key: `verify_code:{phone/email}`, 过期时间5分钟） |
| **认证** | Spring Security + JWT，令牌存储于HttpOnly Cookie |
| **前端组件** | Element Plus（ElForm、ElInput、ElButton、ElMessage等） |

### 3.3 API接口

#### 3.3.1 发送验证码

- **路径**: `POST /api/auth/send-code`
- **请求体**:

```json
{
  "account": "string",
  "type": "string"
}
```

- **参数说明**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| account | string | 是 | 手机号或邮箱 |
| type | string | 是 | `register`/`login`/`reset` |

- **响应**:

```json
{
  "code": 200,
  "message": "验证码发送成功",
  "data": null
}
```

#### 3.3.2 用户注册

- **路径**: `POST /api/auth/register`
- **请求体**:

```json
{
  "account": "string",
  "password": "string",
  "confirmPassword": "string",
  "code": "string"
}
```

- **参数说明**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| account | string | 是 | 手机号或邮箱 |
| password | string | 是 | 密码（≥8位，包含大小写字母和数字） |
| confirmPassword | string | 是 | 确认密码 |
| code | string | 是 | 验证码 |

- **响应**:

```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "token": "string"
  }
}
```

#### 3.3.3 用户登录

- **路径**: `POST /api/auth/login`
- **请求体**:

```json
{
  "account": "string",
  "password": "string",
  "rememberMe": boolean
}
```

- **参数说明**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| account | string | 是 | 手机号或邮箱 |
| password | string | 是 | 密码 |
| rememberMe | boolean | 否 | 是否记住（默认false） |

- **响应**:

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "user": {
      "id": "number",
      "username": "string",
      "avatar": "string",
      "role": "string"
    }
  }
}
```

#### 3.3.4 密码找回

- **路径**: `POST /api/auth/reset-password`
- **请求体**:

```json
{
  "account": "string",
  "code": "string",
  "newPassword": "string"
}
```

- **响应**:

```json
{
  "code": 200,
  "message": "密码修改成功，请重新登录",
  "data": null
}
```

#### 3.3.5 用户退出

- **路径**: `POST /api/auth/logout`
- **响应**:

```json
{
  "code": 200,
  "message": "退出成功",
  "data": null
}
```

### 3.4 安全与隐私

| 安全项 | 措施 |
|--------|------|
| **密码存储** | BCrypt加密存储，禁止明文 |
| **令牌安全** | JWT存入HttpOnly Cookie，防止XSS攻击 |
| **Cookie属性** | `HttpOnly; Secure; SameSite=Strict` |
| **验证码防护** | Redis存储+过期时间+发送频率限制 |
| **暴力破解防护** | 密码错误3次锁定5分钟 |
| **数据传输** | HTTPS加密传输 |
| **敏感信息** | 接口响应不返回密码字段 |

## 5 风险与路线图

### 5.1 分阶段推出

| 阶段 | 内容 | 时间 |
|------|------|------|
| **MVP** | 基础登录注册功能，手机号/邮箱验证，密码找回 | 第1周 |
| **v1.1** | 增加记住我功能，登录状态持久化 | 第2周 |
| **v1.2** | 增加图形验证码，防机器人攻击 | 第3周 |
| **v2.0** | 第三方登录（微信/QQ），账号绑定 | 第2个月 |

### 5.2 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 短信验证码成本 | 大量用户注册导致短信费用增加 | 限制发送频率，优先使用邮箱验证码 |
| 暴力破解攻击 | 账号安全受到威胁 | 密码错误锁定机制，图形验证码 |
| Cookie跨域问题 | 前后端分离架构下Cookie无法正确传递 | 配置正确的CORS策略，使用SameSite属性 |
| 密码找回滥用 | 恶意用户频繁找回密码骚扰 | 限制找回频率，增加安全验证步骤 |

## 6 数据模型

### 6.1 用户表 (users)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | bigint | PRIMARY KEY, AUTO_INCREMENT | 用户ID |
| username | varchar(50) | NOT NULL, UNIQUE | 用户名/账号 |
| password | varchar(255) | NOT NULL | 加密后的密码 |
| email | varchar(100) | NULL, UNIQUE | 邮箱 |
| phone | varchar(20) | NULL, UNIQUE | 手机号 |
