# 智慧图书阅读项目 — 技术架构文档

---

## 1 技术栈概览

### 1.1 整体架构

采用前后端分离架构，前端通过 RESTful API 与后端通信。

| 层级 | 技术选型 |
|------|---------|
| **前端** | Vue 3 + JavaScript + Vite |
| **UI组件库** | Element Plus |
| **图标库** | @element-plus/icons-vue |
| **图表库** | ECharts + vue-echarts |
| **HTTP客户端** | Axios |
| **后端框架** | Spring Boot 4.1.0 |
| **编程语言** | Java 17 |
| **数据库** | MySQL 8.0+ |
| **缓存** | Redis 7.0+ |
| **ORM框架** | MyBatis Plus 3.5.7 |
| **安全框架** | Spring Security |
| **媒体存储** | 腾讯云 COS |
| **认证方案** | HttpOnly Cookie + JWT |

## 3 后端技术架构

### 3.1 框架与语言

| 技术 | 版本 | 用途 |
|------|------|------|
| Spring Boot | 4.1.0 | 后端应用框架 |
| Java | 17 | 编程语言（LTS） |

### 3.2 数据库与缓存

#### MySQL

- **版本**: 8.0+
- **用途**: 持久化存储用户数据、图书信息、阅读记录等
- **配置**:
  - 数据库名：`zhihuitushu`
  - 端口：`3306`
  - 字符集：`UTF-8`
  - 时区：`Asia/Shanghai`

#### Redis

- **版本**: 7.0+
- **用途**:
  - 缓存热点数据（图书信息、用户会话）
  - 存储验证码
  - 分布式锁
- **配置**:
  - 端口：`6379`
  - 数据库：`0`

### 3.3 ORM框架

#### MyBatis Plus

- **版本**: 3.5.7
- **用途**: 简化数据库操作，提供CRUD通用方法
- **核心特性**:
  - 自动生成Mapper代码
  - 分页查询
  - 条件构造器
  - 逻辑删除
  - 驼峰命名转换

### 3.4 安全框架

#### Spring Security

- **用途**: 认证与授权管理
- **认证方案**: HttpOnly Cookie + JWT
  - 用户登录成功后，生成JWT令牌
  - 将JWT存入HttpOnly Cookie返回给前端
  - 前端每次请求自动携带Cookie
  - 后端通过过滤器验证JWT有效性
- **安全特性**:
  - CSRF防护
  - 密码加密（BCrypt）
  - 请求限流

### 3.5 媒体存储

#### 腾讯云 COS

- **版本**: cos_api 5.6.166
- **用途**: 存储图书封面、用户头像等媒体资源
- **配置**:
  - Region: `ap-guangzhou`
  - Bucket: 用户自定义
  - 访问域名: `https://{bucket}.cos.ap-guangzhou.myqcloud.com`
- **核心功能**:
  - 文件上传/下载
  - 文件删除
  - 访问权限控制
  - CDN加速

## 4 核心配置说明

### 4.1 前端Vite代理配置

```javascript

  port: 5173,
  proxy: {
    '/api': {
      target: 'http://localhost:8080',
      changeOrigin: true,
      secure: false,
    },
  },
```

```properties
spring.cors.allowed-origins=http://localhost:5173
spring.cors.allowed-methods=GET,POST,PUT,DELETE,OPTIONS
spring.cors.allowed-headers=*
spring.cors.allow-credentials=true
```

## 5 接口设计规范

### 5.1 请求方法规范

| 场景 | 方法 |
|------|------|
| 单个/无参数查询 | GET |
| 多个参数查询/新增/修改/删除 | POST |

### 5.2 参数封装

- POST请求的多个参数需封装为独立的DTO类
- VO转换逻辑放在独立的converter包中，在ServiceImpl中调用

### 5.3 响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

## 6 安全设计

### 6.1 认证机制

- **令牌类型**: JWT
- **存储方式**: HttpOnly Cookie
- **Cookie属性**: `HttpOnly; Secure; SameSite=Strict`
- **过期策略**: 短时间过期（如1小时），配合刷新机制

### 6.2 密码安全

- 使用BCrypt进行密码加密
- 禁止明文存储密码
- 密码强度校验（长度≥8，包含大小写字母和数字）

### 6.3 数据安全

- 敏感信息传输使用HTTPS
- 用户敏感数据加密存储
- SQL注入防护（MyBatis Plus自动参数化）
